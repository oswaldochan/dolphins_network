Abstract

Network analysis allows us to obtain relevant information about the interactions of a group of individuals.

When these networks are weighted, i.e., each edge has a numerical value associated with it, which in this case is the ratio that two dolphins were seen together, it provides even more information to draw conclusions about their behavior.

For this project we analyzed three networks corresponding to foraging, socializing and traveling in a population of bottlenose dolphins (Tursiops truncatus) in Cedar Key, Florida.

The results coincide with some previous research done on this species, where it has been shown that depending on the type of activity, they prefer to associate with certain dolphins only.

Needless to say, a better understanding of this species can be useful in helping to preserve it.

Introduction

The chosen network corresponds to bottleneck dolphins, which is publicly available in the Animal Social Structure Repository (ASNR).

It should be noted that the data were originally collected and shared publicly by the team formed by Stefanie Gazda, Swami Iyer, Timothy Killingback, Richard Connor and Solange Brault in the paper The importance of delineating networks by activity type in bottlenose dolphins (Tursiops truncatus) in Cedar Key, Florida.

The paper describes the data collection process, where behavioral information was collected from 147 resident dolphins in the Cedar Key, Florida area.

This process was carried out in 2 different periods, which took place in 2008 and 2010.

3 different types of behaviors can be described by the following networks:

  • Social: characterized by repeated incidents of physical contact such as rubbing or petting
  • Travel: characterized by consistent displacement in speed and direction
  • Forage: characterized by the capture or search for prey, defined by specific dolphin movements.

In terms of network type, the 3 networks are weighted undirected, where different ways of defining the weights were included, as it can be seen in the dataset shared in the original paper

On the other hand, given the very nature of the interaction between 2 dolphins, it is assumed that it is not possible for one dolphin to maintain an interaction with another dolphin that is unidirectional.

The graphing and calculations were done on the connected component of the larger network to avoid problems, which resulted in the exclusion of 4 nodes from the forage network and 2 nodes from the travel network.

 

Network Characteristics

The interpretations of the metrics are quite simple, starting with the number of nodes, which corresponds to the number of dolphins in the network, and the number of edges corresponds to the number of dolphins that have interacted with each other.

It is worth mentioning that the value used for the weights corresponds to the Half-Weighted Index (HWI), which is a metric commonly used in research on dolphins and other species. It goes from 0 to 1, and represents how much two individuals were seen together with respect to their total number of observations.

In the case of the average degree, this value represents the average number of dolphins that a dolphin has interacted with.

In the case of the average clustering coefficient, the higher this value is, the more likely it is that a pair of dolphins that interact with another dolphin, also interact between them.

The radius and diameter of each network were also calculated, and correspond to the minimum and maximum number of edges between a specific dolphin and the dolphin farthest from it. This metric could be useful to get an idea of how connected the ends of the network are, for example, in the case of transmitting information.

With respect to the differences between networks, it can be observed that although in theory the observations were made on the same groups of dolphins, they vary in size, probably due to the fact that the researchers established a threshold on the minimum number of interactions that a dolphin should have.

Community detection

Working with weighted networks is in general more complicated than working with unweighted networks, since many of the algorithms were not designed in the first place to work efficiently with weights. In the paper Analysis of weighted networks published by Newman, he mentions some alternatives for working with this type of networks, which can be summarized as follows:

  • Eliminate the weights: this solution is the simplest to use, since the network will be treated as unweighted, so the algorithms will work normally. However, the main problem with this approach is the loss of valuable information provided by the weights about the structure of the network and the interaction between nodes.

  • Converting the network to multigraph: as mentioned in the paper, if we have a network of positive integer weights, the weights could be considered as connections between one node and another, so the number of edges in the multigraph network should be equal to the sum of all the weights in the weighted network. Once the modification is done, you can work with the network and the algorithms in a normal way.

  • Adapting the algorithms: some of the computations can be modified to be useful again on weighted networks, however, in most library implementations, the way in which the weights affect the algorithm are not clear and are not well described. (such as networkx in this case).

    Another type of modification that can be made to the algorithms is to customize them, even if this means modifying the algorithm implementations.

In the aforementioned paper there is a very interesting discussion about the detection of communities in weighted networks.

In summary, in an unweighted network, it is assumed that those edges nodes that act as connections between two communities will have a high betweenness centrality, since many of the communications between communities would have to pass through them following the shortest path.

Thus, by iteratively eliminating these "connector" edges, the communities that make up the network can be discovered.

If we take into account that in the case of weighted networks, the shortest paths between two nodes are the paths where the sum of the weights of the edges is less, it is easy to see why the algorithm mentioned above will fail.

This is because edges that join strongly connected nodes will have a high betweenness centrality, so according to the algorithm , those edges should be eliminated, which is the opposite of what we want, since as it can be seen, we would be breaking the communities and eliminating the edges with the highest interaction between them.

For this problem, the paper proposes a solution by applying the technique of converting the weighted network to a multigraph. Fortunately, it also provides a solution that can be applied directly on the weighted network, modifying the original algorithm:

Another and simpler way of summarizing this algorithm is the following: we calculate the betweennesses of all edges in our weighted graph in the normal way, ignoring the weights. Then we divide each such betweenness by the weight of the corresponding edge, remove the edge with the highest resulting score, recalculate the betweennesses, and repeat.

This can be done with the networkx library, since the implementation of the Girvan-Newman algorithm supports as parameter a function to calculate the most important node, which is a user-defined function that takes as input a network and returns the edge to remove, so it would only be necessary to implement this function as described in the paper.

Conclusions

After having done the analysis of the networks of interactions between dolphins, it is possible to realize that dolphin behavior has a certain level of complexity, because depending on the type of activity they are performing, they behave differently, where in the case of the social network, dolphins behave as a single large group, unlike the other two networks, where dolphins have shown to have preferences and therefore only associate with specific dolphins to do these activities.

Despite the relatively small size of the network, the fact of having worked with weighted networks increased the difficulty of this project, since much of the literature on the main algorithms such as centrality measures, are designed for unweighted networks, although currently many of the social networks that exist are weighted, which shows that it is still a field of study with great potential.

As a personal conclusion, this work was a challenge since it is my first time using the plotly library, which was used in the plotting of the networks, as well as in those of their distributions.
